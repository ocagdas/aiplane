# aiplane v0.1.1 — Developer preview patch

This patch keeps the developer-preview workflow stable while tightening release automation and install-artifact traceability.

## Primary workflow

```bash
aiplane quickstart local-coding --dry-run
aiplane discover
aiplane doctor
aiplane recommend
aiplane export continue
```

Exports print configuration for review; they do not install or edit the target tool.

## Changes

- CI now bumps the patch version after a successful protected-main merge, updates `pyproject.toml` and `src/aiplane/__init__.py`, creates the matching version tag, and avoids recursive version bumps on its own release commits.
- Main-branch wheel artifacts are built from the exact versioned commit after the release gate passes.
- CI wheel artifact names use the version once plus a 7-character commit suffix, for example `aiplane-wheel-v0.1.1-5dab675`.
- Local wheel snapshots can be built into ignored `.aiplane/wheelhouse/` with a PEP 440 local version such as `0.1.1+g5dab675.20260714t153000z`.
- GitHub Actions are updated to maintained Node 24-compatible action majors to avoid Node 20 deprecation warnings.

## Install

Download the wheel and `SHA256SUMS` from this release, verify the checksum, then choose one installation owner:

```bash
uv tool install ./aiplane-0.1.1-py3-none-any.whl
pipx install ./aiplane-0.1.1-py3-none-any.whl
python -m pip install ./aiplane-0.1.1-py3-none-any.whl
```

Package-index installation is not advertised in this preview.

## Integrity

Release automation generates `SHA256SUMS` from the wheel and source distribution after building them and verifies the manifest before publication. Main-branch CI artifacts also include `provenance.json` with the source commit, versioned commit, short versioned commit, release tag, workflow run ID, and wheel filename.

## Upgrade, uninstall, and rollback

Follow the exact commands in [setup](../../user/setup.md#standard-wheel-install-no-repository-clone). Preserve reviewed profile YAML before changing versions. To roll back, uninstall with the same tool that installed Aiplane and reinstall the previously downloaded immutable wheel after verifying its checksum. Published versions are never retagged or silently replaced.

## Developer-preview limitations

This release is pre-1.0 alpha software. External-user demonstrations and the final post-trial documentation sweep remain readiness work. Profiles are not signed, redaction is best effort, and Aiplane is not a secret manager, runtime, model proxy, coding agent, or hidden cloud deployment engine.
